# NyxID Mobile Authenticator（中文汇报版）

## 1. 项目一句话定义

NyxID Mobile Authenticator 是 NyxID 的移动端授权终端，用于接收后端高风险挑战消息（Challenge），并在 App 内完成审批、拒绝、撤销和审计闭环。

## 2. 为什么现在要做

当前 NyxID 高风险操作主要依赖网页端处理，存在三类问题：

1. 用户离开桌面后无法及时处理挑战，业务请求被阻塞。
2. 仅靠轮询触达弱，缺少系统级通知提醒。
3. 授权、撤销、审计链路分散，移动端安全闭环不完整。

本需求的核心价值是：**把“后端触发 -> 用户决策 -> 权限生效 -> 审计留痕”做成移动端可用的标准流程**。

## 3. 目标与边界（MVP）

### 3.1 目标

- 支持 iOS/Android 状态栏推送（APNs/FCM）实时触达。
- 支持通知点击直达 Challenge 详情并完成审批/拒绝。
- 支持授权时长选择、已授权项撤销、审计可追踪。
- 保障关键链路安全（幂等、防重放、最小暴露）与稳定性。

### 3.2 本期范围

- 登录后设备注册与 Push Token 绑定
- Challenge 推送触达与深链跳转
- 审批/拒绝（含时长）
- 已授权列表与撤销
- 审计事件列表与回执

### 3.3 本期不做

- 手表端独立审批
- 短信/语音多通道通知
- 移动端策略编排后台

## 4. 用户流程（业务闭环）

1. 业务服务触发敏感操作，NyxID 创建 Challenge（`PENDING`）。
2. 后端通过 APNs/FCM 向目标设备推送状态栏通知。
3. 用户点击通知进入 App，按 `challenge_id` 拉取详情。
4. 用户执行 Approve/Deny（可选审批时长）。
5. NyxID 更新状态并对下游策略即时生效。
6. 全链路写入审计日志，支持回溯与风控分析。

## 5. 方案亮点（管理层关注）

- **触达能力提升**：从“被动进入 App”升级为“系统级主动提醒”。
- **业务效率提升**：高风险审批可移动端即时处理，降低操作等待时间。
- **安全能力提升**：支持撤销、过期、幂等与审计追踪，降低误授权风险。
- **可运营可观测**：可量化监控推送成功率、审批时延、失败率。

## 6. 核心技术设计（高层版）

### 6.1 后端模块

- `challenge-core`：挑战创建与状态机
- `push-dispatcher`：APNs/FCM 发送、重试、回执
- `decision-handler`：审批/拒绝/撤销接口（幂等）
- `device-registry`：设备与 Token 生命周期管理
- `audit-log`：全链路安全事件记录

### 6.2 客户端模块

- Push 接收与通知路由
- Deep Link 跳转（`nyxid://challenge/{id}`）
- 收件箱/详情/审批/撤销
- 审计记录展示

### 6.3 安全原则

- 推送只放最小字段，不放敏感明文
- 详情信息必须向后端实时拉取
- 决策接口强制状态校验 + 幂等键
- Token 安全存储（Keychain/Keystore）

## 7. 关键指标（KPI / SLA）

- 推送可达率：`>= 98%`
- Challenge 创建到首条推送发出（P95）：`< 2s`
- 通知点击到详情打开（P95）：`< 1.5s`
- 决策接口成功率：`>= 99.9%`
- 安全事故（误授权）：`0 容忍`

## 8. 风险与应对

- 推送不稳定：重试机制 + Inbox 拉取兜底
- Token 频繁变化：自动刷新上报 + 失效清理
- 重复点击并发：前后端双重幂等保护
- 敏感信息泄漏：通知最小化字段策略

## 9. 实施节奏（建议 4 周）

- 第 1 周：数据模型、设备注册、挑战/决策 API
- 第 2 周：APNs/FCM、推送回执、深链打通
- 第 3 周：撤销/审计全链路、异常与安全加固
- 第 4 周：联调压测、灰度发布、监控告警收口
